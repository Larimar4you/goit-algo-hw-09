# goit-algo-hw-09

Висновки:

1. У даній задачі було реалізовано два підходи до видачі решти: жадібний алгоритм
   та алгоритм динамічного програмування.

2. Жадібний алгоритм працює дуже швидко та має лінійну часову складність,
   оскільки на кожному кроці вибирає найбільший можливий номінал монети.
   Для канонічних систем монет (як у даному завданні: 50, 25, 10, 5, 2, 1)
   він завжди знаходить оптимальне рішення.

3. Алгоритм динамічного програмування гарантує знаходження мінімальної кількості
   монет для будь-якого набору номіналів, але має значно більшу часову та
   просторову складність. При великих сумах він працює повільніше та потребує
   більше пам’яті.

4. Отже, для касових апаратів та реальних грошових систем доцільніше
   використовувати жадібний алгоритм, оскільки він є швидшим та достатньо
   ефективним. Алгоритм динамічного програмування варто застосовувати у випадках,
   коли набір номіналів не гарантує оптимального результату жадібного підходу.
